數位醫療的防線：運用區塊鏈技術保護醫療隱私

#Ethereum Attestation Service (EAS) #Blockchain #GiCS 2025

方案緣起

隨著醫療產業數位化，中心化系統逐漸成為駭客目標。2025年3月彰基遭勒索軟體攻擊事件，揭示了醫療系統在權限控管與資料共享上的脆弱性。 本團隊旨在建構一套可即時驗證、去中心化且不可竄改的安全機制，守護病患隱私。

核心解決方案：導入 EAS

  • 獨立證明紀錄： 每次資料存取自動產生不可竄改的上鏈紀錄，確保法律效力。
  • 病患專屬 UID： 透過唯一識別碼綁定醫療紀錄，精準控管授權，防範濫權行為。
  • 彈性隱私選擇： 使用者可決定資料是否上鏈，敏感個資可僅產生離線證明。
  • 去中心化韌性： 運作於以太坊主網，不因單一節點故障影響驗證系統穩定性。

基於深度學習之即時情緒辨識系統於高壓職業環境的應用與評估

#Deep Learning #CNNs & OpenCV #FIDA Algorithm

研究動機與目的

針對警察、醫療等高壓職場環境，現行情緒評估往往缺乏專業即時性。
本研究開發一套即時辨識系統，旨在透過技術干預及預防情緒失衡引發的職業風險。

核心技術實作

  • 影像預處理： 利用 OpenCV 進行臉部聚焦、特徵辨識及光線補償，提升環境適應力。
  • 模型架構： 採用 CNNs 深度學習模型 進行特徵擷取，並結合 FIDA 演算法 進行加權分析。
  • 四大功能模組： 整合情緒辨識（即時反饋）、情緒補給（調節管道）、情感連線（專家疏導）與情緒足跡（個人紀錄）。

研究價值

  • 提升泛化能力： 透過 FIDA 演算法保留關鍵特徵，顯著提升模型在非實驗室環境下的辨識準確度。
  • 學術發表認可： 論文經同儕審查，發表於中國科技大學主辦之國際學術研討會。

私立醫學大學知識轉譯與教育吸引力之評估：產學能量與招生吸引力之對話

#大數據分析 #Data Cleaning #知識轉譯 #Power BI 可視化介面

專案核心

在少子化與高等教育競爭激烈的背景下，本研究聚焦於私立醫學大學如何透過強化產學合作與研發表現，來維持穩定的招生吸引力。
透過整合 106 年至 112 年之校務公開資料，釐清研發能量與招生指標之間的互動機制。

分析維度

  • 產學研發指標： 分析專利申請件數、新品種授權數、以及產學合作計畫之平均經費。
  • 教育吸引力指標： 追蹤歷年新生註冊率、在學學生人數及專任師資變動趨勢。
  • 跨校整合比對： 針對北醫、中醫大、高醫、慈濟、中山醫等多所醫學大學進行數據標準化評比。

技術流程

  • 資料清洗： 處理大專校院校務資訊公開平台之原始數據，包含名稱簡化、學籍編訂與異常值過濾。
  • 關聯性建模： 建立產學合作經費與新生入學意願之間的統計關聯。
  • 數據可視化： 利用直觀圖表（如男女人數比、專利件數趨勢）呈現複雜的教育體系數據。

借塑一宿：封閉式循環區域回收系統建置

#AI 影像辨識 #封閉式循環系統

計畫目標與困境

目前台灣廢塑膠回收率不到三成 ，多數民眾環保意識不足且回收管道便捷性有限。 本團隊提出「借塑一宿」方案，以超商為核心據點，建立封閉式循環系統，最大化塑膠再利用價值。

三大核心模組

  • 回收獎勵機制： 利用超商高密集度特點，民眾回收具企業 LOGO 之塑膠可獲得獎金與環保點數，提升回收意願。
  • 智慧塑膠循環： 利用 AI 辨識回收標誌進行分類，將塑膠再製成食品包裝或台灣特色造型悠遊卡（如藍白拖、紅色塑膠椅）。
  • 互動教育 APP： 整合環保問答小遊戲、電子寵物餵養與點數兌換功能，強化民眾環保意識與參與感。

預期社會效益

  • 環境友善： 透過熱解作用 (pyrolysis) 將無法再製的塑膠轉化為新能源，降低碳排放。
  • 企業責任： 協助企業計算產品回收率，強化綠色形象並落實 ESG 貢獻。

青青眼護你 Glaucosence

#AI 影像分析 #PHP & MySQL #UI/UX Design

專案背景

青光眼是全球第二大致盲原因，台灣患者超過 40 萬人。由於初期症狀不明顯，且需長期甚至終生治療，患者常因忘記服藥或回診而錯失治療時機。

核心功能與技術

  • AI 輔助診斷： 透過 Web 端 AI 模型分析 OCT、眼底及隅角影像，協助醫師精準判斷病情並減少誤診風險。
  • 眼安小博士： APP 內建 VHPA 演算法，根據使用者日常用眼紀錄與護眼數據，以圖表形式分析潛在風險。
  • 治療依從性提升： 提供資料共享與紀錄功能，協助患者定期治療，根據評估可提高約 70% 的治療依從性。
  • 眼保衛教專區： 建立資料大補帖，透過趣味性測驗與獎勵機制，提升大眾對眼部疾病的預防意識。

創新特色

  • 全方位健康監控： 從民眾自測到醫師診斷，建構完整的眼部健康生態系。
  • 智能化回饋： 提供智慧化回覆個人問題，解決醫療資源諮詢不便的痛點。

AI 初步問診系統：智慧醫療服務層架構實作

#Node-RED #LLM Integration #Medical Informatics #UI/UX Design

開發動機與解決問題

針對醫療需求增加導致的候診時間長、民眾缺乏醫療知識容易掛錯科別等問題，本系統旨在建立一個完善的前端症狀評估機制。

核心功能模組

  • 引導式問診： 透過預設提示詞與選項，引導使用者詳細描述感受。
  • AI 症狀分析與風險評估： 根據輸入內容進行分析，並提供可能疾病機率。
  • 推薦就診科別： 整合分析結果，建議最適合的就診科別。
  • 結構化報告生成： 產出問診報告供醫療人員參考，優化診斷流程。

系統架構

  • 前端介面： 提供對話式 UI，讓使用者輸入症狀並即時顯示分析報告。
  • 處理層 (Logic Layer)： 運用 Node-RED 串接 AI 引導、分析、評估與報告生成模組。
  • 服務層整合： 預留與醫院掛號系統及遠距醫療平台串接的介面。

進貨管理系統：資料庫整合應用

#Microsoft SQL Server #CRUD Operations #Data Analysis

專案核心

本系統針對便利商店進貨流程設計，核心在於處理 Convenience Store PO 資料庫中的四個關鍵資料表（商品、供應商、進貨單據、進貨明細）。 透過標準化資料處理流程，確保從前端輸入到後端儲存的資料完整性與安全性。

核心功能與實作

  • 全方位 CRUD 操作： 實現商品與訂單的「新增、查詢、修改、刪除」。系統具備防呆機制，如刪除訂單前會進行二次確認，確保數據安全。
  • 進階查詢系統： 支援依供應商編號（如 S012）快速檢索特定貨單，提升倉儲管理人員的作業效率。
  • 熱門預測功能： 透過歷史數據分析展示商品預測狀況，並主動提示使用者針對特定廠商增減訂貨量，實現資料導向的採購策略。

開發歷程與核心概念

本專案聚焦於後端資料庫的建構基礎與前端網頁資訊層級的配置。
透過 XAMPP 環境，系統性地學習了如何將複雜的資料轉化為結構化的資料庫表單，並嘗試架設類似於維基百科的網站介面。

技術實作亮點

  • 資料庫建置： 使用 phpMyAdmin 建立會員資料、產品規格與訂單分類，練習資料表之間的關聯性設計與 SQL 基本操作（如資料更正與篩選）。
  • 網頁架設： 模擬維基百科的版面佈局，實作側邊導覽列、搜尋框及分類條目，練習 HTML 標籤的語義化運用與 CSS 版面配置。

技術收穫

  • 資料處理能力： 掌握了基本的 CRUD 資料維護邏輯，能精確執行資料的新增、修改與刪除作業。
  • 網站架構思維： 透過模擬 Wiki 系統，理解了網站資訊架構。
